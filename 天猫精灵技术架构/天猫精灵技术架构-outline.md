# 天猫精灵技术架构调研大纲

## 整体架构层次（从底到顶）

```
Layer 7: 应用与生态层     — 技能平台、IoT生态、内容服务、第三方接入
Layer 6: 云端服务层       — AliGenie云平台、阿里云IaaS、边缘计算ENS、知识图谱
Layer 5: AI语音算法层     — 声学唤醒、ASR、NLU/NLG、TTS、对话管理DM
Layer 4: 音频前端处理层   — 波束成形、AEC回声消除、降噪NS、VAD检测
Layer 3: 操作系统层       — AliOS Things、Linux、Android、RTOS微内核
Layer 2: 驱动与固件层     — 音频驱动、WiFi/BT驱动、BSP板级支持
Layer 1: 硬件层           — SoC、麦克风阵列、功放扬声器、无线模组、电源
```

---

## Layer 1：硬件层

### 1.1 主控SoC
- **X1代（2017）**：联发科 MediaTek MT8516，四核 Cortex-A35，带独立NEON处理单元
- **X6/新款**：全志 Allwinner R328-S2，双核 Cortex-A7 @ 1.2GHz，集成语音处理DSP
- **平头哥自研**：玄铁910（RISC-V，2.5GHz，用于部分IoT芯片模组）
- 全志 R328 集成 3 ADC + 1 DAC + 3 I2S + 8 DMIC 音频接口

### 1.2 内存与存储
- 全志R328-S2：64MB DDR2 / R328-S3：128MB DDR3
- 外置SPI NAND Flash 或 NOR Flash 用于系统存储

### 1.3 麦克风阵列
- **X1 早期**：6个MEMS麦克风组成环形阵列（思必驰方案，敏芯微电子MEMS）
- **X6 新款**：3个MEMS麦克风，支持360°拾音 + 硬件VAD
- 5米内远场语音拾取能力

### 1.4 扬声器与功放
- X6功放芯片：ACMESEMI ACM8628 数字功放，Hi-Res Audio认证，THD < 0.03%
- 音腔结构：套筒式独立音腔 + 被动辐射振膜（低频增强）
- 典型功率：5W（方糖系列）/ 更高功率用于X系列

### 1.5 无线通信模组
- Wi-Fi（2.4GHz/5GHz）
- 蓝牙 5.0（BLE + Classic）
- 蓝牙Mesh（接入IoT设备）
- 部分网关版本支持 Zigbee / Matter 协议

### 1.6 电源管理
- 外置电源适配器（如X6：24V/1A）
- 板载电源管理IC

### 1.7 外壳与声学结构
- ABS工程塑料外壳
- 底部橡胶防滑垫
- 声腔调音（低频辐射器）

## Layer 2：驱动与固件层（BSP）

- 音频子系统驱动（ALSA / HAL抽象层）
- Wi-Fi / 蓝牙驱动（RT-Thread / Linux内核模块）
- GPIO、I2C、SPI、UART 等外设驱动
- 板级支持包（BSP）适配全志R328

---

## Layer 3：操作系统层

- **AliOS Things**：阿里自研IoT操作系统，支持RTOS模式、轻量内核模式、纯微内核模式
  - 微内核ROM < 2KB，RAM < 1KB，适合资源受限设备
  - 天猫精灵CC MINI 是首款搭载微内核系统的智能音箱
- **Linux**：用于算力较强的型号
- **Android**：带屏幕型号（CC/CCL系列）
- AliOS Things 已适配 FreeRTOS、RT-Thread、Linux、Android 等主流系统

---

## Layer 4：音频前端处理层（Audio Front End, AFE）

- **波束成形（Beamforming）**：DSB延迟叠加 / MVDR自适应算法，定向增强目标声音
- **回声消除（AEC）**：消除扬声器播放声音对麦克风的干扰
- **噪声抑制（NS）**：多通道维纳滤波，降低环境噪声
- **语音活动检测（VAD）**：硬件VAD（X6内置）实现低功耗常时监听
- 信号增强后输出给语音唤醒和ASR模块

---

## Layer 5：AI语音算法层

### 5.1 语音唤醒（KWS/Wake Word）
- 唤醒词："天猫精灵"
- 多模态唤醒（AliGenie 5.0）：语音 + 唇动 + 手势，满足其中两项即可唤醒
- 误唤醒率通过实验室评测（模拟家庭环境）和线上评测双维度控制

### 5.2 ASR（自动语音识别）
- 端云协同：端侧轻量模型 + 云端高精度模型
- 离线ASR支持（基础指令）+ 在线ASR（复杂语义）
- 算法通过亿万句真实对话打磨

### 5.3 NLU（自然语言理解）
- 两级模型架构：**领域分类模型DC** + **联合意图分类与实体抽取模型ICSF**
- 意图（Intent）、实体（Entity）、参数（Slot）三层解析
- 上下文管理：多轮对话跟踪

### 5.4 对话管理（DM, Dialog Manager）
- 对话状态跟踪（DST）
- 策略决策（Policy）
- 响应生成（NLG）

### 5.5 TTS（语音合成）
- 基于深度学习的端到端TTS，支持多音色
- 使用NVIDIA TensorRT + Triton 加速推理，提升流式合成效率
- 支持语速、音量调节
